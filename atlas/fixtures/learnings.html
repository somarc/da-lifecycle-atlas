A new site is not proof because the boilerplate cloned. Every friction between empty repository and verified public page belongs here.

The first finding arrived before the first write.

The repository and AEM Code Sync were healthy, but the cached DA IMS token expired before content installation. The CLI reported the exact binary provenance, the invalid auth state, and executable recovery. That is a useful Wave 6 baseline, not an inconvenience to hide.

Static beauty is not content-driven proof.

The first iteration served a polished standalone HTML explainer from the codebus. It proved design and Code Sync, but it had no editable DA narrative. The correction is now explicit: DA owns content; blocks own presentation; Git may keep the original visual as a design reference, not the canonical page.

The first code branch name was not a valid EDS ref.

The initial branch used slashes. da-cli warned that its normalized host label would not correspond to a real Git ref and that code operations would 404. The branch was renamed to feature-wave-6-full-da-site before construction. The public code asset then returned 200 before any content mutation began.

Riverboat carried Grok Imagine without pretending it was safe or deterministic.

The constrained pipeline first produced an unsafe execution plan with one shell step and one explicit approval gate. After human approval, Riverboat invoked a tool-filtered Grok Build 0.2.102 session, captured one 1280×720 image, hashed the binary, prompt, wrapper, YAML, output, and image, and left the result pending review. Human review promoted that exact file; the normal DA pipeline then uploaded the frozen asset. The rendered page serves an optimized 750×422 derivative with no about:error.
